# Pipeline Core — Slash commands

_This file is GENERATED by `scripts/generate-slash-docs.mjs` during install (and on demand via `make pipeline-generate-docs`). It exists so `pipeline-drift-scan` can verify every enabled slash command is documented. Edit `scripts/lib/slash-commands.mjs` upstream, then regenerate._

All commands below are invoked by posting a comment on a GitHub issue or PR. The comment-commands workflow parses the body and dispatches the command if the actor and state allow it.

## Commands

| Command | Moves issue to | Allowed actors | Notes |
| --- | --- | --- | --- |
| `/ready` | Selected for Development | human | Move from Triage to Selected for Development if the ready-gate passes. |
| `/needs-info` | Needs Info | human | Move to Needs Info; comment captures what's missing. |
| `/info-resolved` | Triage | human, agent | Return to Triage from Needs Info. |
| `/block` | Blocked | human, agent | Move to Blocked; reason captured in a comment. |
| `/unblock` | __previous | human, agent | Restore previous state. |
| `/park` | On Hold | human, agent | Park (On Hold) with revisit date or condition. |
| `/unpark` | __previous | human, agent | Restore previous state. |
| `/refuted` | Done | human, agent | Close issue; hypothesis refuted; applies `refuted` label. |
| `/duplicate` | Done | human, agent | Close as duplicate of another issue (e.g. /duplicate #42). |
| `/wontfix` | Done | human, agent | Close: won't fix. |
| `/cnr` | Done | human, agent | Close: cannot reproduce. |
| `/reopen` | Triage | human | Reopen a closed issue; routes back to Triage by default, or Selected for Development with --skip-triage. |
| `/grab` | In Progress | human, agent | Claim issue; assign self; move to In Progress. |
| `/release` | Selected for Development | human, agent | Release issue back to Selected for Development (caller must be current assignee). |
| `/iterate` | In Progress | human, agent | Start a new iteration from In Review. Caller must be assignee or have triage permission. |
